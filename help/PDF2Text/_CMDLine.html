Command Line

Command line specification

The application may be run directly from the command line, without GUI.
The format of command line is:

PDF2Text Pilot.exe ((SrcFile (DstFile)) | (/inlist SrcListFile /outlist DstListFile) | (/inlist SrcListFile /outdir DstDir)) [options]

i.e.:

PDF2Text Pilot.exe SrcFile DstFile [options]
or
PDF2Text Pilot.exe /inlist SrcListFile /outlist DstListFile [options]
or
PDF2Text Pilot.exe /inlist SrcListFile /outdir DstDir [options]

Where:

SrcFile - the path to a PDF file to be converted to text file. It can be absolute (ex: "C:\documents\some.pdf") or relative (ex: "some.pdf")

DstFile - the path to a created text file. If it is omitted then txt extension is added to SrcFile (ex: if SrcFile is "C:\test.pdf" then DstFile is set to "C:\test.pdf.txt"). It can also be absolute (ex: "C:\Documents\books\some_book.txt") or relative (ex: "some_book.txt")

SrcListFile - the path to a file that contains a list of files to be converted to text files. It can also be absolute or relative. The file must contain one file name per line. Here is the content of such file, for example:

DstListFile - the path to a file that contains a list of file names to be saved as. It can also be absolute or relative. The file must contain one file name per line. Here is the content of such file, for example:



Example of DstListFile content.

Please note: The number of files listed in SrcListFile MUST match the number of files listed in DstListFile.

DstDir - the path to a directory where created files will be saved to. It can also be absolute or relative. All new file names will be the same as the source file names, and have the extension .txt. (ex: "C:\Documents\books\TXT")

options - the list of secondary parameters that allows you to set detailed properties of a created document.

These parameters are:

/all2one - set this parameter if you want to obtain one text file from all input files.
/password value - set this parameter if PDF document is protected.
/author - the text files will have the same settings with the PDF file.
/subject - the text files will have the same settings with the PDF file.
/title - the text files will have the same settings with the PDF file.
/keywords - the text files will have the same settings with the PDF file

Returned codes

-1 Command line format error. Occurs when parameters are set incorrectly.
-2 Can't open the PDF file.
-3 The output path is not correct.
-4 The output file name is not correct.
-5 Out-folder is not found.
-7 Can't open the input list file.
-10 Can't open the output list file.
-13 Input list does not match the output list. Occurs when the number of files specified in the input list is not equal to the number of files specified in the output list.

Command line examples

  • PDF2Text Pilot.exe "D:\Documents\my.pdf" "D:\Documents\my.txt"

    Convert the file D:\Documents\my.pdf into text file with default settings and save it as D:\Documents\my.txt.


  • PDF2Text Pilot.exe "thePDF.pdf" "theTXT.txt" /author /subject /title /password "pass"

    Convert the file "thePDF.pdf" into text file and save it as "theTXT.txt. Sets the author, subject, and title in text file. The document "thePDF.pdf" is protected. It can be opened using the password "pass".


  • PDF2Text Pilot.exe /inlist src.lst /outdir "D:\docs\TXT

    Convert the PDF files listed in "src.lst" into text files and saves them into directory "D:\docs\TXT".
